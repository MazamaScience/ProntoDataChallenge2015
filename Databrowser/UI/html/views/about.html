About this Databrowser

Goals

This databrowser attempts to make it easy for anyone with a question about Pronto Cycle usage to answer that question quickly and easily with self-explanatory, publication-ready graphics.

In public discourse, people often make confident statements about an issue without consulting available data. With disturbing regularity, many of these confident statements are "demonstrably false" after one finds and analyzes relevant data.

By familiarizing youself with the actual bikeshare trip data you can decide for yourself how well it is working and how/where it should be modified or expanded. (Perhaps underperforming stations should be moved?)

Software

All calculations and data visualizations are performed with the R statistical programming language. The user interface was built with AngularJS and Boostrap. A python CGI script connects requests from the user interface to R code using the python rpy2 module. All communication between the client and server is via http GET using JSON.

Full, ugly, unrefactored, seat-of-the-pants source code is available on github at: https://github.com/MazamaScience/ProntoDataChallenge2015.

Data

Trip, Station and Weather data are all available from Pronto's Data Challenge page. Additional information on elevation was obtained from the Google Maps Elevation API. Ephemerides calculations (dawn/dusk) are provided by the maptools package in R.
